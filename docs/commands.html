All commands run from your regular terminal. There is no REPL — tccp gives you a persistent shell on the compute node instead.

session lifecycle

commandwhat it does
tccp startFull startup: allocate a SLURM node, pull container, sync files, run init, start persistent shell via dtach. Requires tccp.yaml in the current directory.
tccp shellAttach to the persistent shell. You're inside the container on the compute node with GPU access. Ctrl+S detaches, syncs, and reattaches. Ctrl+D exits.
tccp stopPull output, cancel the SLURM job, clear session state.
$ tccp start
$ tccp shell
tccp> python train.py
tccp> exit
$ tccp stop

sync & exec

commandwhat it does
tccp syncPush changed files to compute node + pull output back to laptop. Also happens automatically on Ctrl+S during tccp shell.
tccp exec <cmd>Run a one-off command inside the container on the compute node and print the result. Useful for quick checks without attaching to the shell.
$ tccp sync
$ tccp exec "python -c 'import torch; print(torch.cuda.is_available())'"

info

commandwhat it does
tccp statusShow session info: job ID, compute node, scratch path, container, start time, ports.
tccp allocsList your SLURM allocations with job ID, name, partition, GPU, time, and state.

GPU resources

commandwhat it does
tccp gpusLive GPU availability across all partitions. Shows available vs total count per GPU type.
tccp gpus <type>Per-node breakdown for a specific GPU type. e.g. tccp gpus a100
tccp gpus infoStatic GPU guide with VRAM, use cases, and recommendations. See also GPUs page.

housekeeping

commandwhat it does
tccp dealloc <id>Cancel a SLURM allocation by job ID.
tccp dealloc allCancel all your SLURM allocations.
tccp setupSave or update your cluster credentials (username and password).
