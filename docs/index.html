why this exists

I just wanted to code. Open my editor, change a file, run it on a GPU. The same loop I'd have locally. I didn't want to write SLURM scripts, scp files around, manage containers, debug module loads, do quota math, or get Duo-pushed into oblivion.

The cluster should be invisible. Type one command, your code runs on a GPU. That was the whole idea.

tccp sits between your laptop and the compute node and handles all the stuff in between. You just write code.

what it handles for you

  • Connects with a single SSH session and a single Duo push
  • Syncs your code to the compute node when you run a job
  • Pulls and caches Singularity containers as needed
  • Sets up Python virtualenvs and installs your requirements.txt
  • Finds or allocates SLURM compute nodes (and reuses them across jobs)
  • Lets you attach and detach from running jobs, like tmux
  • Forwards ports for things like TensorBoard and Jupyter
  • Brings your output files back when the job finishes

quick start

# install the binary (see install page)

# save your Tufts credentials once
$ tccp setup

# set up your project
$ cd my-project
$ tccp register

# connect and run
$ tccp
tccp> run train
