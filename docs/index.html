motivation

I wanted to write ML code like I write any other code. Locally. Open my editor, change a file, run it. No SLURM scripts, no scp, no ssh tunnels, no quota math, no environment debugging.

The cluster should be invisible. I should not have to think about allocations, Singularity containers, scratch directories, NFS mounts, or Duo pushes. I should type one command and my code runs on a GPU.

tccp abstracts away the compute. You code locally. tccp handles everything between your laptop and the GPU.

what it does

  • One SSH connection, one Duo push per session
  • Syncs your code to the compute node automatically
  • Pulls and caches Singularity containers
  • Creates and manages Python virtualenvs
  • Allocates and reuses SLURM compute nodes
  • Attach/detach from running jobs (like tmux)
  • Port forwarding for TensorBoard, Jupyter, etc.
  • Auto-retrieves output when jobs finish

quick start

# 1. Install (see install page)

# 2. Store your Tufts credentials
$ tccp setup

# 3. Register your project
$ cd my-project
$ tccp register

# 4. Connect and run
$ tccp
tccp> run train
